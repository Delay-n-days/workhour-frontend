# 功能码编辑器优化计划

**日期：** 2026-06-04
**目标：** 优化功能码编辑器，使其更适合从 Excel 迁移的用户

---

## 一、需求总结

### 核心约束
1. **功能码（function_code）不可编辑** - 这是唯一标识符
2. **组号/分组（group）不可编辑** - 分组是固定的结构
3. **不需要批量编辑** - 用户在 Excel 中也不这么做

### 用户场景
- 客户从 Excel 迁移过来
- 需要能查看所有参数
- 普通模式下只修改出厂值
- 高级模式下可以修改其他参数（除功能码和分组）

---

## 二、当前问题分析

### 问题 1：表格列不完整
当前只显示：属性、功能码、注释、出厂值、单位、范围
缺少：分组、地址、精度、选项说明、显示格式、数据宽度

### 问题 2：编辑方式不直观
当前：点击行 → 打开对话框 → 修改
期望：双击单元格直接编辑

### 问题 3：对话框太宽泛
当前：所有字段都在对话框中
期望：普通模式只显示出厂值输入框

---

## 三、实现方案

### 方案概述
1. **表格显示所有参数列** - 让用户一目了然
2. **表格内直接编辑** - 双击出厂值列直接编辑
3. **对话框只用于高级模式** - 修改除功能码和分组外的其他参数
4. **功能码和分组永远不可编辑** - 在任何地方都显示为只读

### 步骤 4：保存状态指示

**行状态样式：**
```tsx
// 行样式
const getRowClassName = (fc: FuncCode, index: number) => {
  if (editingCell?.index === index) return "bg-blue-50" // 编辑中
  if (modifiedRows.has(fc.function_code)) return "bg-yellow-50 border-l-2 border-l-yellow-400" // 已修改
  return ""
}
```

**保存反馈：**
- 保存成功：行背景短暂变绿（500ms）
- 保存失败：行背景变红 + 显示错误提示

### 步骤 5：禁止编辑功能码和分组

**在对话框中：**
```tsx

// 分组 - 始终只读

  <SelectValue />
</Select>
```

**在表格中：**
- 功能码和分组列不响应点击事件
- 不显示编辑图标
